Troubleshooting

Common issues and their solutions, organized by tool.

Build Issues

gcc: error: gtk+-3.0 not found

GTK3 development headers are missing. Install them:

sudo apt install libgtk-3-dev
pkg-config --libs gtk+-3.0 fails

Ensure pkg-config is installed and GTK3 .pc files are on the path:

sudo apt install pkg-config libgtk-3-dev
pkg-config --modversion gtk+-3.0   # should print e.g. 3.24.33
Permission denied when running build scripts

Make the build scripts executable:

chmod +x build_*.sh

Cursor Spotlight

Spotlight window doesn't appear

Check X11 composite extension is enabled. Spotlight requires compositing for transparency:

# Verify compositing
xdpyinfo | grep Composite

# If missing, ensure a compositor is running (picom, compton, xcompmgr)
picom --daemon
Spotlight appears but no transparency

Your compositor may not support ARGB visuals. Try:

# Start with explicit 32-bit visual
./cursor_spotlight --visual 32

# Or check picom configuration
picom --backend glx --daemon
Spotlight flickers or has wrong position

Multi-monitor setups may cause offset issues. Specify the target display:

./cursor_spotlight --display :0 --screen 0

Gromit-MPX

F6 does nothing / Gromit not responding

Gromit-MPX may not be running. Start it first:

gromit-mpx --key F6 --undo-key F5 &
# Then press F6 to toggle drawing mode

If it's running but unresponsive, check for zombie processes:

pkill -f gromit-mpx
gromit-mpx --key F6 --undo-key F5 &
Drawing strokes are invisible

This usually means the compositor isn't providing transparency. Solutions:

  • Ensure a compositor is running (picom, xcompmgr)
  • Check Gromit's tool config: ~/.config/gromit-mpx.cfg
Custom Gromit profile not loading

Run the installer script to set up the Antsand Gromit profile:

./install_gromit_profile.sh
gromit-mpx -q >/dev/null 2>&1 || true
gromit-mpx --key F6 --undo-key F5 >/dev/null 2>&1 &

This copies the custom tool definitions to ~/.config/gromit-mpx.cfg.

Shift drag does not switch to expected shape

Verify the profile is installed and Gromit was restarted after installation:

./install_gromit_profile.sh
gromit-mpx -q >/dev/null 2>&1 || true
gromit-mpx --key F6 --undo-key F5 >/dev/null 2>&1 &

Expected profile mappings include: Shift marker, Ctrl line, Alt rectangle, Alt+Shift circle, Alt+Ctrl filled circle.

Gromit-MPX does not provide a text tool.

Presenter Dash

"xdotool: command not found" 
sudo apt install xdotool
Segments draw in wrong position

The anchor point may be stale. Reset and re-anchor:

./presenter_dash.sh reset
./presenter_dash.sh anchor

Control Center

Night Light toggle has no effect

Ensure redshift is installed:

sudo apt install redshift
# Test manually
redshift -O 3500   # warm
redshift -x         # reset
Window appears but tabs are empty

GTK theme may be missing. Install a complete theme package:

sudo apt install adwaita-icon-theme-full gnome-themes-extra

AwesomeWM

Config fails to load / black screen after restart
  1. Switch to TTY: Ctrl+Alt+F2
  2. Check the error: awesome --check ~/.config/awesome/rc.lua
  3. Restore backup: cp ~/.config/awesome/rc.lua.bak ~/.config/awesome/rc.lua
  4. Return to GUI: Ctrl+Alt+F7
  5. Reload: awesome-client 'awesome.restart()'
Hotkeys conflict with other applications

Check for conflicts in AwesomeWM's help popup (Mod4+s). Common conflicts:

  • F6 / F11 — some browsers use these for address bar / fullscreen
  • Solution: use the keys only when Gromit is active, or remap in rc.lua
rescue-windows.sh not working

The script uses wmctrl. Install it:

sudo apt install wmctrl

GTK DSL

Parser error: "unknown widget type"

Check spelling of the widget type. Valid types: box, grid, label, button, switch, scale, entry, image.

Widget doesn't appear in window

Every widget (except root) needs a parent property pointing to a valid id:

# Wrong — no parent
label text="Hello"

# Right
label parent=root text="Hello"

General

How to check all dependencies at once 
# Quick dependency check
for cmd in gcc pkg-config gromit-mpx xdotool redshift picom wmctrl awesome; do
    printf "%-15s" "$cmd"
    if command -v "$cmd" >/dev/null 2>&1; then
        echo "✓ $(command -v $cmd)"
    else
        echo "✗ NOT FOUND"
    fi
done
How to report a bug

Include the following in your report:

uname -a
awesome --version
pkg-config --modversion gtk+-3.0
echo $XDG_SESSION_TYPE
