One-Line Install

sudo apt install build-essential pkg-config \
    libx11-dev libxext-dev libxfixes-dev libxrender-dev libcairo2-dev \
    libgtk-4-dev gromit-mpx xdotool
./install_gromit_profile.sh   # installs compat profile by default

Building Tools

Cursor Spotlight

./build_cursor_spotlight.sh
./build/bin/cursor_spotlight --radius 180 --dim 0.68 --fps 50
# Optional install
./build_cursor_spotlight.sh --install
~/Programs/bin/cursor_spotlight --radius 180 --dim 0.68 --fps 50

Press Esc to exit. See Spotlight docs for all flags.

Linux Control Center

./build_linux_control_center.sh
./build/bin/linux_control_center
# Optional install
./build_linux_control_center.sh --install
~/Programs/bin/linux_control_center

GTK4 app with screenshot browser, multi-select controls, and annotation canvas.

GTK DSL Demo

./build_gtk_dsl_demo.sh
./build/bin/gtk_dsl_demo dsl/workbench.gdsl
# Optional install
./build_gtk_dsl_demo.sh --install
~/Programs/bin/gtk_dsl_demo dsl/workbench.gdsl

GTK4 DSL runtime. Edit dsl/workbench.gdsl to customise the launcher.

Presenter Dash

# No build step — it's a shell script
./install_gromit_profile.sh   # install parser-safe profile once
./presenter_dash.sh anchor   # set anchor at cursor
./presenter_dash.sh dash     # dashed segment
./presenter_dash.sh arrow    # arrow segment
./launch_teleprompter.sh     # local script reader window

ATEM setup: keep teleprompter window on laptop panel and HDMI output for the captured app windows.

AwesomeWM Configuration

ℹ
Use the Make targets so rc.lua and the linuxutils/ module tree stay in sync.
make awesome-user-update

# Reload AwesomeWM
# Press Mod4 + Ctrl + R   (or restart from the menu)

The config adds a Utilities widget to the wibar with buttons for Spotlight, Gromit, Control Center, and shortcuts. All presenter-related hotkeys are on the F6–F11 row.

Building the Docs

cd docs
bash build.sh

# Preview locally
cd build
python3 -m http.server 8080
# Open http://localhost:8080

Generated pages go into docs/build/ which is gitignored. Only _pages/, _partials/, css/, js/, and assets/ are tracked in version control.
