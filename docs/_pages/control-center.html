Control Center

A native GTK4 desktop utility focused on screenshot workflow: browse images, filter/select/delete quickly, and annotate in-place using event-controller driven canvas tools.

Build & Run

cd LinuxUtilities
./build_linux_control_center.sh   # compiles linux_control_center.c
./build/bin/linux_control_center  # launch the app
# Optional install
./build_linux_control_center.sh --install
~/Programs/bin/linux_control_center

Recommended launch paths:

# From repo (always uses newest local build)
./build/bin/linux_control_center ~/Workspace/LinuxUtilities

# Installed binary (global launch)
~/Programs/bin/linux_control_center ~/Workspace/LinuxUtilities

Or compile manually:

mkdir -p build/bin
gcc -o build/bin/linux_control_center linux_control_center.c \
    $(pkg-config --cflags --libs gtk4) -lm

Current GTK4 Features

🖼 Screenshot Browser

Loads images from Screenshots/, shows thumbnails in a multi-select grid, and supports instant search filtering.

⌨️ Keyboard Workflow

Ctrl+A selects all thumbnails, Esc clears selection, and Delete removes selected files.

✏️ Annotation Canvas

Select an image to annotate with Select, Arrow, Line, Rect, Callout, Text, and Step tools. Uses GTK4 event controllers for click/drag/move/scroll handling.

🔍 Zoom + File Ops

Ctrl+Scroll zooms the canvas. Toolbar actions include refresh, open folder, and delete selected.

🎙️ Audio + Mic Helpers

Audio tab includes quick actions for Bluetooth mic mode (HFP/HSP), music mode (A2DP), status dump, and microphone help.

🎛️ Style Ribbons

Top tool row and quick-style row are horizontally scrollable, with per-tool property controls in the right dock.

UI Troubleshooting

Toolbar icons missing or style ribbon not visible

Most often this is a stale binary launch issue. Rebuild and run a known binary path:

cd ~/Workspace/LinuxUtilities
./build_linux_control_center.sh --install
pkill -f linux_control_center || true
~/Programs/bin/linux_control_center ~/Workspace/LinuxUtilities

If rows are still hidden, reset saved UI layout state:

mv ~/.config/linuxutilities/control_center_gtk4.ini \
   ~/.config/linuxutilities/control_center_gtk4.ini.bak.$(date +%s)

AwesomeWM Integration

Launch the Control Center from AwesomeWM with a hotkey:

-- rc.lua
awful.key({ modkey }, "c", function()
    awful.spawn("linux_control_center")
end, {description = "open control center", group = "utilities"})
